Building a “browser-like iframe sandbox” in Node is a fun but tricky problem, because Node’s vm module gives isolation of scope, but not full security by itself. To make it closer to a real sandbox, you combine:

vm → separate execution context
Proxy → intercept dangerous global access
Object.freeze → prevent mutation of APIs
a carefully controlled global object

Below is a practical, modern pattern.

🧱 1. Goal: what we’re building

We want a sandbox where user code:

❌ cannot access process, require, fs
❌ cannot escape to Node globals
❌ cannot modify host objects
✅ can only use a limited API (like browser iframe window)
🧠 2. Core idea

We create:

“fake window”
const sandbox = {
  console,
  Math,
  Date,
};

Then:

freeze everything inside
wrap in Proxy
run inside vm
🔐 3. Secure sandbox implementation
const vm = require('vm');

/**
 * Create a locked-down sandbox environment
 */
function createSandbox() {
  const sandbox = {
    console: Object.freeze({
      log: (...args) => console.log('[sandbox]', ...args),
      error: (...args) => console.error('[sandbox]', ...args),
    }),

    Math: Object.freeze(Math),
    Date,
  };

  // Freeze top-level object
  Object.freeze(sandbox);

  return sandbox;
}
🧲 4. Proxy layer (prevents escaping attempts)
function createSecureProxy(context) {
  return new Proxy(context, {
    get(target, key) {
      if (key === 'global' || key === 'globalThis') {
        return undefined;
      }

      return target[key];
    },

    set(target, key, value) {
      if (target[key] && Object.isFrozen(target[key])) {
        throw new Error(`Cannot modify frozen property: ${String(key)}`);
      }
      target[key] = value;
      return true;
    },

    has(target, key) {
      // hides Node internals
      if (['process', 'require', 'eval', 'Function'].includes(key)) {
        return false;
      }
      return key in target;
    },
  });
}
⚙️ 5. Create VM context
function runInSandbox(code) {
  const sandbox = createSandbox();
  const proxySandbox = createSecureProxy(sandbox);

  const context = vm.createContext(proxySandbox, {
    name: 'secure-sandbox',
  });

  const script = new vm.Script(code);

  return script.runInContext(context, {
    timeout: 1000,
    displayErrors: true,
  });
}
🚀 6. Example usage
runInSandbox(`

  console.log(x);

⚠️ 8. Important truth (security reality)

Even with vm + Proxy + freeze:

👉 Node vm is NOT fully secure against determined attackers

Because:

prototype pollution tricks
context leaks via Error objects
exotic JS engine behaviors
