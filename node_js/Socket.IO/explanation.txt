1️⃣ Importing Modules
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

What each does:

express → creates a web application

http → creates a raw HTTP server

Server from socket.io → enables real-time communication

path → helps safely build file paths

2️⃣ Creating App and Server
const app = express();
const server = http.createServer(app);
const io = new Server(server);
What happens here?

app → Express app

server → HTTP server wrapping Express

io → Socket.IO attached to the HTTP server

⚠️ Important:
Socket.IO must attach to the HTTP server, not directly to Express.

3️⃣ Serving Static Files
app.use(express.static(path.join(__dirname, 'public')));

This tells Express:

“Everything inside the public folder can be accessed directly by the browser.”

Example:

public/script.js → accessible

public/index.html → accessible

4️⃣ Route for /
app.get('/', (req, res) => {
  res.sendFile(path.join(__dirname, 'public', 'index.html'));
});

When someone visits:

The server sends:

public/index.html

1️⃣ socket.emit() vs io.emit()

Inside:

io.on('connection', (socket) => {

You get a socket object that represents ONE connected user.

🔹 socket.emit()

Sends message to ONLY that specific user

socket.emit('chat message', 'Hello only you');

✔ Only the connected user receives it.

🔹 io.emit()

Sends message to EVERYONE

io.emit('chat message', 'Hello everyone');

✔ All users receive it
✔ Including the sender

🔹 socket.broadcast.emit()

Sends to everyone except sender

socket.broadcast.emit('chat message', 'Someone joined');

✔ Everyone sees it
❌ Sender does NOT

2️⃣ How Frontend Connects to Server

Inside public/index.html:

<script src="/socket.io/socket.io.js"></script>
<script>
  const socket = io();

  socket.on('chat message', (msg) => {
    console.log(msg);
  });

  function sendMessage() {
    const input = document.getElementById('m');
    socket.emit('chat message', input.value);
    input.value = '';
  }
</script>
What happens:

Browser loads Socket.IO client

io() connects automatically to server

You send message with:

socket.emit('chat message', message);

Server receives it

Server broadcasts it

All clients receive it instantly

That’s real-time WebSocket communication powered by Socket.IO.

3️⃣ Add Usernames 🧑
Step 1 — Ask for username when connecting

Server:

io.on('connection', (socket) => {
  socket.on('set username', (username) => {
    socket.username = username;
  });

  socket.on('chat message', (msg) => {
    io.emit('chat message', {
      user: socket.username,
      text: msg
    });
  });
});

Frontend:

const username = prompt("Enter your name:");
socket.emit('set username', username);

Now messages look like:

{
  user: "Andrey",
  text: "Hello"
}
4️⃣ Chat Rooms 🏠

Socket.IO supports rooms built-in.

Join a room:
socket.on('join room', (room) => {
  socket.join(room);
});
Send message to room only:
socket.on('chat message', ({ room, msg }) => {
  io.to(room).emit('chat message', msg);
});

Frontend:

socket.emit('join room', 'room1');
socket.emit('chat message', {
  room: 'room1',
  msg: 'Hello room1'
});

Now only users in room1 receive it.

5️⃣ Private Messages 🔐

Each socket has a unique ID:

console.log(socket.id);
Send private message:
socket.on('private message', ({ to, msg }) => {
  io.to(to).emit('private message', {
    from: socket.id,
    text: msg
  });
});

Frontend:

socket.emit('private message', {
  to: 'TARGET_SOCKET_ID',
  msg: 'Hello privately'
});

Only that user receives it.
